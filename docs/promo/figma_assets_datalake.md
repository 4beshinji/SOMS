### 見出しテキスト (セクション別)

#### 問題提起
- 都市データの根本的矛盾: 活用にはプライバシーの犠牲が必要とされてきた
- スマートシティはデータを集めるほど、住民のプライバシーリスクが増大する。Data Clean Roomも「データを1箇所に集める」構造は変わらない。

#### 解決策: 分散プライバシー保護
- 生データを動かさない。ノイズを加えた統計だけを共有する。
- 各建物のCore Hubが差分プライバシー (ε-DP) を適用し、数学的に証明可能なプライバシー保証の下で都市データレイクを構築する。

#### 3層プライバシーアーキテクチャ
- Layer 1: ローカル処理 — 生データは建物内で完結。映像はRAM処理後に即時破棄。
- Layer 2: リージョンハブ — Secure Aggregationで暗号化集約。個別Hubの値は復元不可。
- Layer 3: 都市アグリゲーター — 統計のみ受領。個人・建物単位のデータは一切到達しない。

#### ESP32エッジプライバシー
- プライバシー保護はクラウドではなく、センサーの隣で行う
- ESP32上でLocal DPノイズを注入。生の測定値がデバイスを離れる前にプライバシーが確保される。

#### 法的コンプライアンス
- 個人データを収集しないため、GDPRの同意要件・個人情報保護法の適用対象外
- Privacy by Design / by Default を技術的に実装。法的義務ではなく、アーキテクチャが保証する。

#### 都市の呼吸パターン
- プライバシーを完全に保護しながら、都市のマクロパターンを可視化する
- CO2集約値だけで人流パターンが浮かび上がる。誰がどこにいたかは、誰にもわからない。

### 本文テキスト (コンポーネント用)

#### エレベーターピッチ (50字)
差分プライバシーで生データを建物内に封じ込め、統計だけで都市を理解するデータレイク。

#### 短い説明 (100字)
SOMS Data Lakeは、各建物のCore Hubが差分プライバシー (ε-DP) を適用し、生データを一切外部送信せずに都市規模の統計データレイクを構築する。Secure Aggregationと暗号化で数学的プライバシー保証を実現。個人情報保護法・GDPRの同意要件が構造的に不要。

#### 詳しい説明 (200字)
SOMS Data Lakeは、プライバシー保護と都市データ活用の二律背反を解消するアーキテクチャ。各建物のCore HubがセンサーデータをLocal DPで処理し、Secure Aggregationプロトコルでリージョンハブに暗号化集約。都市アグリゲーターは統計曲線のみを受領し、個別建物の値すら復元できない。ESP32エッジデバイス上でLaplaceノイズを注入するため、生測定値がネットワークに出ることはない。50,000:1のデータ圧縮と数学的プライバシー保証 (ε = 1.0) を両立。クラウド費用$0、同意取得コスト$0、プライバシーリスク0の都市データ基盤。

### 特徴カード用テキスト (6枚)

**カード1: Local Differential Privacy**
タイトル: ローカル差分プライバシー
アイコン: シールド + ε
本文: ESP32センサーノード上でLaplaceノイズを注入。生の測定値がデバイスを離れる前に数学的プライバシー保護を適用。データ収集者すら真の値を知ることができない。

**カード2: Secure Aggregation**
タイトル: 暗号化集約
アイコン: 鍵 + 合流する矢印
本文: 複数Core Hubの値をSecure Aggregationプロトコルで集約。合計値のみが復号され、個別Hubの寄与分は数学的に復元不可能。

**カード3: ε Budget Management**
タイトル: プライバシー予算管理
アイコン: メーター + ε
本文: 各デバイス・各チャネルにε予算を設定。予算を超えるクエリは自動拒否。時間経過で予算が回復するスライディングウィンドウ方式。

**カード4: Legal Compliance**
タイトル: 構造的法令遵守
アイコン: チェックマーク + 法律書
本文: 個人データを収集しない設計のため、GDPR同意要件・個人情報保護法の規制対象外。Privacy by Designをアーキテクチャレベルで実装。

**カード5: ESP32 Compatible**
タイトル: ESP32エッジ対応
アイコン: チップ + 盾
本文: 240MHz・520KB RAMのESP32上でDP処理が完結。高価なサーバーなしに、$5のマイコンでプライバシー保護を実現。SensorSwarmの全Leafノードに展開可能。

**カード6: Urban Patterns**
タイトル: 都市パターン抽出
アイコン: 都市 + 波形グラフ
本文: プライバシーを完全保護しながら、CO2・温湿度・占有率の集約パターンから都市の「呼吸」を可視化。個人の行動は一切追跡しない。
